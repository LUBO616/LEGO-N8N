PROBLEMA: n8n EACCES /home/node/.n8n/config
SÍNTOMA (logs):
Error: EACCES: permission denied, open '/home/node/.n8n/config'

CAUSA:
La carpeta del host ./data/n8n quedó con dueño/permisos incorrectos (muy común si al inicio corriste docker con sudo).
Dentro del contenedor n8n corre como usuario "node" (UID 1000) y necesita escribir en /home/node/.n8n.

SOLUCIÓN:
1) Baja el stack:
./scripts/stack.sh down basic

2) Arregla dueño de data/n8n:
sudo chown -R 1000:1000 ./data/n8n

3) Sube otra vez:
./scripts/stack.sh up basic

4) Verifica logs:
docker logs -n 50 lego-n8n
