PROBLEMA: Variables not set (POSTGRES_*, N8N_ENCRYPTION_KEY) aunque ya tengas .env
SÍNTOMA:
- WARN: The "POSTGRES_USER" variable is not set. Defaulting to a blank string.
- WARN: The "N8N_ENCRYPTION_KEY" variable is not set...

CAUSA:
Si usas "-f compose/core.yaml", Docker Compose a veces toma como "project directory" la carpeta "compose/"
y busca el .env en compose/.env, no en la raíz.

SOLUCIÓN RECOMENDADA (usa project-directory + env-file):
docker compose --project-directory . --env-file .env -f compose/core.yaml config

MEJOR AÚN (usa el script):
./scripts/stack.sh ps basic
./scripts/stack.sh up basic

NOTAS:
- El .env real NO se sube a GitHub.
- El ejemplo es env/core.env.example
